一个工作台，不止会“回答”。

Vela 的重点不是把模型包成聊天框，而是把 mission、记忆、工具行动、权限边界和评审闭环放到一个可操作的桌面里。

Mission Workspace

任务第一屏

主区域服务当前 mission，承载目标、计划、当前步骤、证据、风险和下一步，不把用户推回杂乱面板。

  • 中心工作区保持唯一主表面
  • 右侧 Intelligence Spine 默认折叠
  • 阶段状态和证据同步呈现
Memory System

记忆证据链

记忆召回不是黑箱提示词，而是可以展示来源、置信度、使用原因和对当前 mission 的影响。

  • SQLite + FTS5 持久化
  • 人物记忆、任务知识、行为约束
  • 支持 provenance 和 recall 评估
Action Layer

受控工具行动

工具调用、权限模式和复核声明在同一套 mission runtime 中运行，关键动作有边界、有日志、有回放。

  • send_message / fetch_url
  • read_file / write_file
  • exec_command / permission guard
Voice Layer

语音操作入口

语音不只是输入文本，还要覆盖打断、修正、状态回读、权限确认和任务续跑。

Observability

轨迹与评审

Golden trace、截图检查、产品契约和评审声明让 Vela 的每一步能被验证，而不是只看最终文本。

Compatibility

保留旧能力

兼容 Brain UI、本地 API、语音、工具市场和多模型 provider，Vela Shell 在已有能力之上继续演进。

围绕 mission 组织起来的工作架构。

这不是一个把 prompt 塞给模型的薄壳，而是一条清晰的运行链路：任务定义、上下文装配、权限判断、工具执行、评审写回、下一步继续。

Mission
Goal + Constraints

目标、验收标准、权限和上下文边界先进入 mission runtime。

Workspace
Focused Surface

主工作区只服务当前任务，右侧智能脊柱默认收起，必要时展开查看证据。

Action
Guarded Tool Loop

工具行动由权限模式和风险提示约束，结果回流到任务轨迹和评审区域。

Memory
Recall + Provenance

可解释地召回、引用和写回长期记忆，供后续 mission 继续使用。

Review
Trace + Evidence

把内部状态暴露为可以看见、可以调试、可以继续扩展的界面与接口。

几分钟内把它跑起来。

Vela 的定位是本地可运行、可观察、可继续改造的 AI Operating Desk。你可以先打开 Vela Shell，再按自己的工作流继续扩展它。

OPENAI_API_KEY=your_key_here
